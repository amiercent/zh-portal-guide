# zh-portal-guide

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态形式存在，不依赖后端服务，可直接在浏览器中打开或通过 GitHub Pages 等静态托管服务访问。

## 项目简介

zh-portal-guide 是一个轻量级的 HTML 页面集合仓库。其目的是将分散的、独立的功能页面统一存放，便于版本管理、内容更新与分发。仓库本身不针对特定网站或服务，仅作为页面归档的载体。

## 目录结构

```
zh-portal-guide/
├── index.html          # 仓库首页 / 入口页面
├── pages/              # 存放各独立 HTML 页面的目录
│   ├── page-a.html
│   ├── page-b.html
│   └── ...
├── assets/             # 公共资源（可选）
│   ├── css/
│   ├── js/
│   └── images/
├── README.md           # 本文件
└── LICENSE             # 许可证文件
```

- `index.html`：可选的主页，可作为导航或聚合页面。
- `pages/`：核心目录，每个 `.html` 文件为一个独立页面。
- `assets/`：存放页面共用的样式、脚本或图片资源。

## 页面归档说明

- 每个 HTML 页面应保持独立，尽量不依赖外部资源（或仅依赖本仓库内的公共资源）。
- 页面内容与结构自包含，便于复制、修改或直接部署。
- 归档的页面不强制要求风格统一，但建议保持基本的可读性与兼容性。
- 不建议在页面中包含外部跟踪脚本、第三方动态内容或敏感信息。

## 维护说明

- 本仓库欢迎通过 Pull Request 提交新的页面或对现有页面进行改进。
- 提交新页面时，请确保其功能完整、无明显错误，并放置在 `pages/` 目录下。
- 若需修改公共资源（`assets/` 内文件），请评估对已存在页面的影响。
- 仓库维护者会定期检查页面可用性，但不对页面内容的具体准确性负责。
- 所有贡献均需遵循仓库采用的许可证（见 LICENSE 文件）。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/zh-portal-guide.git
   ```
2. 直接用浏览器打开 `pages/` 下的任意 HTML 文件，或通过本地服务器预览。
3. 也可将仓库部署至 GitHub Pages 或其他静态托管平台，获得在线访问地址。

## 贡献指南

- 请确保新增或修改的页面不违反任何法律法规及平台政策。
- 提交前请测试页面在常见浏览器（Chrome、Firefox、Edge 等）下的基本显示与功能。
- 代码风格无强制要求，但建议保持整洁、注释清晰。
- 提交信息应简要描述变更内容。
